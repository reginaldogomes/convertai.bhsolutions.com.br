# Checklist RAG + SEO para Indexacao

## Objetivo

Usar a base de conhecimento (RAG) para gerar paginas e copys mais relevantes para o nicho e publico-alvo, melhorando qualidade de conteudo e sinais de indexacao organica.

## Checklist tecnico

1. Garantir que entradas da base de conhecimento tenham contexto de negocio:
- nicho
- publico-alvo
- termos de intencao
- localidade (quando aplicavel)

2. Validar retrieval com filtros de relevancia:
- brandName
- niche
- targetAudience
- intentKeywords

3. Reforcar prompts de geracao com contexto SEO:
- topico principal
- publico-alvo
- palavras-chave de intencao
- orientacao para evitar keyword stuffing

4. Validar estrutura de pagina para indexacao:
- hero com proposta clara
- secoes de prova (beneficios, social proof)
- FAQ com duvidas reais
- CTA final coerente com intencao

## Checklist de publicacao

1. Meta title unico por pagina.
2. Meta description com promessa clara + contexto do nicho.
3. Keywords coerentes com intencao (sem repeticao artificial).
4. Canonical configurado quando houver variacoes de URL.
5. Open Graph preenchido (titulo, descricao, imagem).
6. Pagina publica acessivel sem bloqueio acidental de robots.

## Checklist de monitoramento

1. Medir impressões e CTR por pagina.
2. Medir consultas de cauda longa relacionadas ao nicho.
3. Revisar paginas com baixa impressao e reotimizar copy com novo contexto RAG.
4. Atualizar base de conhecimento mensalmente (novas dores, objecoes, casos, diferenciais).

## Cadencia recomendada

1. Semanal: revisar novas entradas de conhecimento.
2. Quinzenal: revisar performance organica das paginas principais.
3. Mensal: atualizar termos de intencao por nicho e ajustar prompts.
